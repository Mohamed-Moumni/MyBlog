FluentPython - DataModel

2026-03-152 min read (222 words)
#Programming  #Languages  #systems  #Python 

Every Python object consists of specific methods called magic methods or dunder methods. These methods allow your custom objects to interact with fundamental language constructs like collections, iteration, and operator overloading.

For example: len

The len function is the result of implementing the len magic method.

import collections

Card = collections.namedtuple('Card', ['rank', 'suit'])


class FrenchDeck:
    ranks = [str(n) for n in range(2, 11)] + list('JQKA')
    suits = 'spades diamonds clubs hearts'.split()

    def __init__(self):
        self._cards = [Card(rank, suit) for suit in self.suits
                                        for rank in self.ranks]

    def __len__(self):
        return len(self._cards)

    def __getitem__(self, position):
        return self._cards[position]


deck = FrenchDeck()

print(len(deck)) ## 52

print(deck[2]) ## Card(rank='4', suit='spades')

Those methods are meant to be called by the Python interpreter, not by the programmer. If you implement the len method, you should call len(my_obj) directly — it will automatically invoke len through the interpreter.

This implementation also helps you leverage the standard library. If you implement getitem and len, your class will automatically become compatible with Python’s standard library.
